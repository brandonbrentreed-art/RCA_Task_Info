## 3. Version Cache-Busting

**Status: ✅ DONE — Content hashes auto-generated by `build.py`.**

**What was done (session 2)**
- `build.py` computes MD5 hash (first 6 chars) of each local CSS/JS file at build time
- All `<link href>` and `<script src>` attributes in built output get `?v=<hash>` appended automatically
- Hash only changes when file content changes — no manual version management needed
- Built pages written to `dist/` — source pages in root remain clean templates

---

## 4. JSON Data Files — Fetch Timing Safety

**Status: ✅ DONE — Audited. All call sites confirmed safe.**

**Audit result (session 2)**
All `_techDB.lookup()` / `_dm.lookup()` call sites are triggered by user actions (paste, file upload, button click), not at page init. `NdpData.restore()` reads from sessionStorage only — no lookup calls. The `ready()` callbacks on both loaders exist as a safety net and no wrapping is required.
